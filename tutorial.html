Understanding Clustering in Self-Organizing Maps Using the Iris Dataset

Introduction | Dataset | SOM Algorithm | Implementation | Results | Analysis | Ethical AI | Conclusion | References

1. Introduction

Modern datasets often contain many variables, making them difficult to interpret and visualize. Traditional visualization methods such as scatter plots become limited when working with high-dimensional data. Machine learning techniques can help reveal hidden patterns and structures that are not easily visible using conventional methods.

Self-Organizing Maps (SOM), introduced by Teuvo Kohonen, are a type of unsupervised neural network that maps high-dimensional data onto a two-dimensional grid. The SOM preserves topological relationships between data points, meaning that similar samples are placed close together on the grid. This property allows researchers to visualize complex datasets and identify clusters or patterns in the data.

In this tutorial, we demonstrate how Self-Organizing Maps can be used to understand clustering behaviour in data. Using the classic Iris dataset, we train a SOM and explore several visualization tools including the U-Matrix, label overlays, and component planes. These visualisations help explain how the SOM organizes data and which features influence the clustering structure.

2. Dataset Description

The Iris dataset is one of the most widely used datasets in machine learning. It contains 150 samples of iris flowers belonging to three species:

  • Iris setosa
  • Iris versicolor
  • Iris virginica

Each sample includes four numerical features:

  • Sepal length
  • Sepal width
  • Petal length
  • Petal width

These measurements allow machine learning algorithms to identify patterns and relationships between the species.

3. Self-Organizing Map Algorithm

A Self-Organizing Map consists of a grid of neurons, where each neuron contains a weight vector of the same dimension as the input data. During training, the SOM learns to represent the structure of the dataset by adjusting these weight vectors.

Key Concepts

Best Matching Unit (BMU)
For each input sample, the neuron whose weight vector is closest to the input vector is selected. This neuron is called the Best Matching Unit.

Neighbourhood Learning
When the BMU is updated, nearby neurons on the grid are also adjusted. This neighbourhood update helps preserve the topology of the data.

Learning Rate
The learning rate controls how much weights change during training and gradually decreases as training progresses.

Training Process

  1. Initialize neuron weight vectors randomly.
  2. Select a sample from the dataset.
  3. Find the Best Matching Unit (BMU).
  4. Update the BMU and its neighbouring neurons.
  5. Decrease learning rate and neighbourhood radius.
  6. Repeat for many iterations until convergence.

The SOM weight update rule is:

w_i(t+1) = w_i(t) + α(t) * h_ci(t) * (x − w_i(t))

4. Implementation in Python

We implement the SOM using the MiniSom Python library. The grid size is set to 7×7 neurons to allow clusters to emerge during training. 


from minisom import MiniSom
from sklearn.datasets import load_iris
import numpy as np

data = load_iris()
X = data.data
y = data.target

som = MiniSom(x=7, y=7, input_len=4, sigma=1.0, learning_rate=0.5)

som.random_weights_init(X)
som.train_random(X, 100)

5. U-Matrix Visualization

The U-Matrix (Unified Distance Matrix) visualizes distances between neighbouring neurons in the SOM grid. Large distances between neurons often indicate cluster boundaries.

6. Label Overlay

By plotting the class labels on the SOM grid, we can see how different species cluster together. If the SOM has successfully captured the structure of the dataset, samples from the same class will appear close together.

7. Component Planes

Component planes display how each feature is distributed across the SOM grid. By comparing component planes, we can identify which variables influence clustering.

8. Analysis

The SOM organizes the Iris dataset into meaningful clusters. The species Iris setosa forms a clearly separated group, while Iris versicolor and Iris virginica partially overlap. This reflects the fact that the two species share more similar measurements.

The component planes reveal that petal length and petal width strongly influence the clustering structure. Sepal measurements contribute less to the separation of species.

9. Ethical Considerations

Although Self-Organizing Maps are unsupervised algorithms, they can still reflect biases present in the data used to train them. When applied to sensitive domains such as healthcare, hiring, or finance, clusters discovered by machine learning models may influence important decisions.

It is therefore important to interpret results carefully, evaluate data sources critically, and communicate uncertainty when presenting clustering results.

10. Conclusion

Self-Organizing Maps provide an intuitive way to visualize and analyze high-dimensional data. In this tutorial, we used SOM to explore clustering behaviour in the Iris dataset and demonstrated several visualization tools including the U-Matrix and component planes.

These techniques help researchers better understand how datasets are structured and how individual features contribute to clustering patterns.
